Figuras en Python - MATPLOTLIB¬∂

Introducci√≥n¬∂

Matplotlib es una excelente biblioteca de gr√°ficos 2D y 3D para generar figuras cient√≠ficas. Algunas de las muchas ventajas de esta biblioteca incluyen:

  • F√°cil de empezar, y cuenta con un gran soporte para etiquetas y textos con formato \(\LaTeX\)

  • Gran control de cada elemento de una figura, incluidos el tama√±o de la figura y el DPI.

  • Salida de alta calidad en muchos formatos, incluidos PNG, PDF, SVG, EPS y PGF.

  • GUI para explorar de forma interactiva los valores en una figura.

Una de las caracter√≠sticas clave de matplotlib que me gustar√≠a enfatizar, y que creo que hace que matplotlib sea muy adecuado para generar figuras para publicaciones cient√≠ficas, es que todos los aspectos de la figura pueden controlarse mediante programaci√≥n. Esto es importante para la reproducir resultados y es conveniente cuando se necesita regenerar la figura con datos actualizados o cambiar su apariencia.

Se puede encontrar una gran cantidad de informaci√≥n al respecto en la p√°gina web de Matplotlib.

Para comenzar a utilizar Matplotlib en un programa de Python, incluya los s√≠mbolos del m√≥dulo pyplot (de manera sencilla):

import matplotlib.pyplot as plt
import matplotlib
# importamos ademas la librer√≠a de calculo num√©rico numpy
import numpy as np

Figuras de matplotlib como objetos¬∂

Comenzamos creando una figura en la variable fig, y a partir de ella creamos una instancia de ejes o axes usando el m√©todo add_axes:

fig = plt.figure() # creamos la figura y la guardamos en fig
axes = fig.add_axes([0.1, 0.1, 0.8, 0.8]) # izquierda, inferior, ancho, alto (rango 0 a 1)
x = np.linspace(-1,1,51)
y = x**2
axes.plot(x, y, 'r') # dibujamos una l√≠nea roja
axes.set_xlabel('x') # etiqueta eje x
axes.set_ylabel('y') # etiqueta eje y
axes.set_title('title'); # titulo de la figura

Notar que tenemos el control total de d√≥nde se colocan los ejes de trazado. Podemos agregar f√°cilmente m√°s de un eje a la figura:

fig = plt.figure()
axes1 = fig.add_axes([0.1, 0.1, 0.8, 0.8]) # eje principal
axes2 = fig.add_axes([0.2, 0.5, 0.4, 0.3]) # eje interior

# figura principal
axes1.plot(x, y, 'r')
axes1.set_xlabel('x')
axes1.set_ylabel('y')
axes1.set_title('title')

# figura interior
axes2.plot(y, x, 'g')
axes2.set_xlabel('y')
axes2.set_ylabel('x')
axes2.set_title('insert title');

Si no nos importa ser expl√≠citos acerca de d√≥nde se ubican nuestros ejes de trazado en la figura, entonces podemos usar uno de los muchos administradores de dise√±o de ejes en matplotlib. Mi favorito es subplots, que se puede usar as√≠:

fig, axes = plt.subplots()

axes.plot(x, y, 'r')
axes.set_xlabel('x')
axes.set_ylabel('y')
axes.set_title('title');

fig, axes = plt.subplots(nrows=1, ncols=2)

for ax in axes:
    ax.plot(x, y, 'r')
    ax.set_xlabel('x')
    ax.set_ylabel('y')
    ax.set_title('title')

Eso fue f√°cil, pero no resulta del todo bonito ya que las etiquetas se superponen a los ejes.

Podemos lidiar con eso usando el m√©todo fig.tight_layout, que ajusta autom√°ticamente las posiciones de los ejes en la ventana de la figura para que no haya contenido superpuesto:

fig, axes = plt.subplots(nrows=1, ncols=2)

for ax in axes:
    ax.plot(x, y, 'r')
    ax.set_xlabel('x')
    ax.set_ylabel('y')
    ax.set_title('title')
    
fig.tight_layout()

Tama√±o de la figura, relaci√≥n de aspecto y DPI¬∂

Matplotlib permite especificar la relaci√≥n de aspecto, el DPI y el tama√±o de la figura cuando se crea el objeto de la clase Figure, utilizando los argumentos claves figsize y dpi. figsize es una tupla del ancho y alto de la figura en pulgadas, y dpi es la cantidad de pixels por pulgada. Para crear una figura de 800x400 pixels, 100 puntos por pulgada (DPI), podemos hacer:

fig = plt.figure(figsize=(8,4), dpi=100);

<Figure size 800x400 with 0 Axes>

Los mismos argumentos tambi√©n se pueden pasar a los gestores de dise√±o de figuras , como la funci√≥n subplots:

fig, axes = plt.subplots(figsize=(12,3))

axes.plot(x, y, 'r')
axes.set_xlabel('x')
axes.set_ylabel('y')
axes.set_title('title');

Guardando figuras¬∂

Para guardar una figura en un archivo, podemos utilizar el m√©todo savefig en la clase Figure:

fig.savefig("filename.png")

Aqu√≠ tambi√©n podemos especificar opcionalmente el DPI y elegir entre diferentes formatos de salida:

fig.savefig("filename.png", dpi=200)

Formatos disponibles: ¬øcu√°les deben usarse para obtener la mejor calidad?¬∂

Matplotlib puede generar resultados de alta calidad en varios formatos, incluidos PNG, JPG, EPS, SVG, PGF y PDF. Para art√≠culos cient√≠ficos, recomiendo usar PDF siempre que sea posible. (Los documentos LaTeX compilados con pdflatex pueden incluir archivos PDF usando el comando includegraphics). En algunos casos, PGF tambi√©n puede ser una buena alternativa.

Leyendas, etiquetas y t√≠tulos¬∂

Ahora que hemos cubierto los conceptos b√°sicos de c√≥mo crear una figura y agregar ejes a la misma, veamos c√≥mo decorar una figura con t√≠tulos, etiquetas de ejes y leyendas.

T√≠tulos de las figuras¬∂

Se puede agregar un t√≠tulo a cada instancia de eje en una figura. Para establecer el t√≠tulo, use el m√©todo set_title en la instancia de los ejes:

ax.set_title("title");

Nombres de eje¬∂

De manera similar, con los m√©todos set_xlabel y set_ylabel, podemos establecer los nombres o etiquetas de los ejes X e Y:

ax.set_xlabel("x")
ax.set_ylabel("y");

Leyendas¬∂

La mejor manera de agregar una leyenda es haciendo uso de la palabra clave label="label text" cuando se agregan sub-figuras u otros objetos a los ejes, y luego usar el m√©todo legend sin argumentos para agregar la leyenda a la figura:

fig, ax = plt.subplots()
ax.plot(x, x**2, label="curva1")
ax.plot(x, x**3, label="curva2")
ax.legend();

La ventaja de este m√©todo es que si se agregan o eliminan curvas de la figura, la leyenda se actualiza autom√°ticamente en consecuencia.

La funci√≥n legend toma un argumento de palabra clave opcional loc que se puede usar para especificar d√≥nde se dibujar√° la leyenda en la figura. Los valores permitidos de loc son c√≥digos num√©ricos para los distintos lugares donde se puede dibujar la leyenda. Consulte la documentaci√≥n de Matplotlib para obtener m√°s informaci√≥n.

Algunos de los valores m√°s comunes de loc son:

ax.legend(loc=0) # matplotlib decide la ubicaci√≥n √≥ptima
ax.legend(loc=1) # arriba a la derecha
ax.legend(loc=2) # arriba a la izquierda
ax.legend(loc=3) # abajo a la izquierda
ax.legend(loc=4); # abajo a la derecha
# .. many more options are available

En el siguiente ejemplo se muestra c√≥mo usar el t√≠tulo de la figura, las etiquetas de los ejes y las leyendas descritas anteriormente:

fig, ax = plt.subplots()

ax.plot(x, x**2, label="y = x**2")
ax.plot(x, x**3, label="y = x**3")
ax.legend(loc=2); # arriba a la izquierda 
ax.set_xlabel('x')
ax.set_ylabel('y')
ax.set_title('title');

Formato de texto: LaTeX, tama√±o de fuente, familia de fuentes¬∂

La figura anterior es funcional, pero (a√∫n) no cumple los criterios de una figura utilizada en una publicaci√≥n. Primero y ante todo, necesitamos tener texto con formato LaTeX, y segundo, debemos poder ajustar el tama√±o de la fuente para que aparezca en una publicaci√≥n.

Matplotlib tiene un gran soporte para LaTeX. Todo lo que tenemos que hacer es usar signos de d√≥lar para encapsular LaTeX en cualquier texto (leyenda, t√≠tulo, etiqueta, etc.). Por ejemplo, "$y=x^3$".

Pero aqu√≠ podemos encontrar un problema ligeramente sutil con el c√≥digo LaTeX y las cadenas de texto de Python. En LaTeX, frecuentemente usamos la barra invertida en los comandos, por ejemplo \alpha para producir el s√≠mbolo \(\alpha\). Pero la barra invertida ya tiene un significado en las cadenas de Python (el car√°cter del c√≥digo de escape). Para evitar que Python estropee nuestro c√≥digo de l√°tex, necesitamos usar cadenas de texto ¬´sin procesar¬ª. Las cadenas de texto sin formato se a√±aden con una ‚Äúr‚Äù, como r"\alpha" o r'\alpha' en lugar de "\alpha" o '\alpha':

fig, ax = plt.subplots()

ax.plot(x, x**2, label=r"$y = \alpha^2$")
ax.plot(x, x**3, label=r"$y = \alpha^3$")
ax.legend(loc=2) # upper left corner
ax.set_xlabel(r'$\alpha$', fontsize=18)
ax.set_ylabel(r'$y$', fontsize=18)
ax.set_title('title');

Tambi√©n podemos cambiar el tama√±o de fuente global y la familia de fuentes, que se aplica a todos los elementos de texto en una figura (etiquetas y t√≠tulos de ejes, leyendas, etc.):

# Actualiza los par√°metros de configuraci√≥n de Matplotlib 
matplotlib.rcParams.update({'font.size': 18, 'font.family': 'serif'})

fig, ax = plt.subplots()

ax.plot(x, x**2, label=r"$y = \alpha^2$")
ax.plot(x, x**3, label=r"$y = \alpha^3$")
ax.legend(loc=2) # upper left corner
ax.set_xlabel(r'$\alpha$')
ax.set_ylabel(r'$y$')
ax.set_title('title');

Una buena elecci√≥n de fuentes globales son las fuentes STIX:

matplotlib.rcParams.update({'font.size': 18, 'font.family': 'STIXGeneral', 'mathtext.fontset': 'stix'})

fig, ax = plt.subplots()

ax.plot(x, x**2, label=r"$y = \alpha^2$")
ax.plot(x, x**3, label=r"$y = \alpha^3$")
ax.legend(loc=2) # upper left corner
ax.set_xlabel(r'$\alpha$')
ax.set_ylabel(r'$y$')
ax.set_title('title');

Configuraci√≥n de colores, tipos de l√≠nea y marcadores¬∂

Colores¬∂

Con matplotlib se puede definir los colores de las lineas y de otros elementos gr√°ficos de varias maneras. Primero podemos usar una sintaxis parecida a la de MATLAB donde 'b' significa blue (azul), 'g' significa green (verde), etc. El API de MATLAB para seleccionar estilos de lineas tambi√©n es soportado: donde 'b.-' significa azul con linea estilo linea punto.

# MATLAB style line color and style 
ax.plot(x, x**2, 'b.-') # blue line with dots
ax.plot(x, x**3, 'g--') # green dashed line

[<matplotlib.lines.Line2D at 0x7fda5442cfa0>]

Tambi√©n podemos definir los colores por sus nombres o c√≥digos hexadecimales RGB y, opcionalmente, proporcionar un valor alfa utilizando los argumentos de las palabras clave color y alpha:

fig, ax = plt.subplots()

ax.plot(x, x+1, color="red", alpha=0.5) # transparencia media
ax.plot(x, x+2, color="#1155dd")        # c√≥digo hexadecimal RGB para un color azulado
ax.plot(x, x+3, color="#15cc55");        # # c√≥digo hexadecimal RGB para un color verdoso

Estilos de l√≠neas y marcadores¬∂

Para cambiar el ancho de l√≠nea, podemos usar el argumento de la palabra clave linewidth o lw. El estilo de l√≠nea se puede seleccionar usando los argumentos de la palabra clave linestyle o ls:

fig, ax = plt.subplots(figsize=(12,6))

ax.plot(x, x+1, color="blue", lw=0.25) # lw=linewidth
ax.plot(x, x+2, color="blue", lw=0.50)
ax.plot(x, x+3, color="blue", lw=1.00)
ax.plot(x, x+4, color="blue", lw=2.00)

# posibles opciones de tipos de linea ‚Äò-‚Äò, ‚Äò--‚Äô, ‚Äò-.‚Äô, ‚Äò:‚Äô, ‚Äòsteps‚Äô
ax.plot(x, x+5, color="red", lw=2, ls='-')
ax.plot(x, x+6, color="red", lw=2, ls='-.')
ax.plot(x, x+7, color="red", lw=2, ls=':')

# trazo a tramos personalizado
line, = ax.plot(x, x+8, color="black", lw=1.50)
line.set_dashes([5, 10, 15, 10]) # formato: largo de linea, largo del espacio, ...

# s√≠mbolos de marcadores posibles: marker = '+', 'o', '*', 's', ',', '.', '1', '2', '3', '4', ...
ax.plot(x, x+ 9, color="green", lw=2, ls='--', marker='+')
ax.plot(x, x+10, color="green", lw=2, ls='--', marker='o')
ax.plot(x, x+11, color="green", lw=2, ls='--', marker='s')
ax.plot(x, x+12, color="green", lw=2, ls='--', marker='1')

# tama√±o y color de los marcadores
ax.plot(x, x+13, color="purple", lw=1, ls='-', marker='o', markersize=2)
ax.plot(x, x+14, color="purple", lw=1, ls='-', marker='o', markersize=4)
ax.plot(x, x+15, color="purple", lw=1, ls='-', marker='o', markersize=8, markerfacecolor="red")
ax.plot(x, x+16, color="purple", lw=1, ls='-', marker='s', markersize=8, 
        markerfacecolor="yellow", markeredgewidth=2, markeredgecolor="blue");

Control sobre la apariencia del eje¬∂

La apariencia de los ejes es un aspecto importante de una figura que a menudo necesitamos modificar para hacer una publicaci√≥n con calidad gr√°fica. Necesitamos poder controlar d√≥nde se colocan las marcas y las etiquetas, modificar el tama√±o de la fuente y posiblemente las etiquetas utilizadas en los ejes. En esta secci√≥n veremos c√≥mo controlar esas propiedades en una figura de matplotlib.

Rango¬∂

Lo primero que podr√≠amos querer configurar es el rango de los ejes. Podemos hacerlo utilizando los m√©todos set_ylim y set_xlim en el objeto de eje, o axis('tight') para obtener rangos de ejes ¬´ajustados¬ª de manera autom√°tica:

fig, axes = plt.subplots(1, 3, figsize=(12, 4))

axes[0].plot(x, x**2, x, x**3)
axes[0].set_title("rangos default de los ejes")

axes[1].plot(x, x**2, x, x**3)
axes[1].axis('tight')
axes[1].set_title("Ejes ajustados 'apretados' axes")

axes[2].plot(x, x**2, x, x**3)
axes[2].set_ylim([0, 60])
axes[2].set_xlim([2, 5])
axes[2].set_title("Rango de los ejes personalizados");

Escala logar√≠tmica¬∂

Tambi√©n es posible establecer una escala logar√≠tmica para uno o ambos ejes. De hecho, esta funcionalidad es solo una aplicaci√≥n de un sistema de transformaci√≥n m√°s general en Matplotlib. Cada una de las escalas de los ejes se establece por separado utilizando los m√©todos set _xscale y set_yscale que aceptan un par√°metro (con el valor ¬´log¬ª en este caso):

fig, axes = plt.subplots(1, 2, figsize=(10,4))
      
axes[0].plot(x, x**2, x, np.exp(x))
axes[0].set_title("Normal scale")

axes[1].plot(x, x**2, x, np.exp(x))
axes[1].set_yscale("log")
axes[1].set_title("Logarithmic scale (y)");

Ubicaci√≥n de las etiquetas y etiquetas personalizadas¬∂

Podemos determinar expl√≠citamente d√≥nde queremos que los tics del eje con set_xticks y set_yticks, que toman una lista de valores para los lugares donde se colocar√°n los tics en el eje. Tambi√©n podemos usar los m√©todos set_xticklabels y set_yticklabels para proporcionar una lista de etiquetas de texto personalizadas para cada ubicaci√≥n de marca:

fig, ax = plt.subplots(figsize=(10, 4))

ax.plot(x, x**2, x, x**3, lw=2)

ax.set_xticks([-1, -0.5, 0, 0.5, 1])
ax.set_xticklabels([r'$\alpha$', r'$\beta$', r'$\gamma$', r'$\delta$', r'$\epsilon$'], fontsize=18)

yticks = [-2, -1, 0, 1, 2]
ax.set_yticks(yticks)
ax.set_yticklabels(["$%.1f$" % y for y in yticks], fontsize=18); # usa latex para formatear los tics

Hay una serie de m√©todos m√°s avanzados para controlar la colocaci√≥n de marcas mayores y menores en las cifras de matplotlib, como la ubicaci√≥n autom√°tica de acuerdo con diferentes pol√≠ticas. Consulte la documentaci√≥n de matplotlib para obtener m√°s informaci√≥n.

Notaci√≥n cient√≠fica¬∂

Con grandes n√∫meros en los ejes, a menudo es mejor usar la notaci√≥n cient√≠fica:

fig, ax = plt.subplots(1, 1)
      
ax.plot(x, (5*x)**2, x, np.exp(5*x))
ax.set_title("Notaci√≥n cient√≠fica")

ax.set_yticks([0, 50, 100, 150])

from matplotlib import ticker
formatter = ticker.ScalarFormatter(useMathText=True)
formatter.set_scientific(True) 
formatter.set_powerlimits((-1,1)) 
ax.yaxis.set_major_formatter(formatter)

N√∫mero de eje y espaciado de la etiqueta del eje¬∂

# distancia entre x e y y el numero del eje
matplotlib.rcParams['xtick.major.pad'] = 5
matplotlib.rcParams['ytick.major.pad'] = 5

fig, ax = plt.subplots(1, 1)
      
ax.plot(x, (5*x)**2, x, np.exp(5*x))
ax.set_yticks([0, 50, 100, 150])

ax.set_title("label and axis spacing")

# separaci√≥n entre la etiqueta del eje y los n√∫meros del eje 
ax.xaxis.labelpad = 5
ax.yaxis.labelpad = 5

ax.set_xlabel("x")
ax.set_ylabel("y");

# Recuperamos los valores por defecto
matplotlib.rcParams['xtick.major.pad'] = 3
matplotlib.rcParams['ytick.major.pad'] = 3

Ajustes de posici√≥n del eje¬∂

Desafortunadamente, al guardar figuras, las etiquetas a veces se recortan, y puede ser necesario ajustar un poco las posiciones de los ejes. Esto se puede hacer usando subplots_adjust:

fig, ax = plt.subplots(1, 1)
      
ax.plot(x, (5*x)**2, x, np.exp(5*x))
ax.set_yticks([0, 50, 100, 150])

ax.set_title("title")
ax.set_xlabel("x")
ax.set_ylabel("y")

fig.subplots_adjust(left=0.15, right=.9, bottom=0.1, top=0.9)

Grillas¬∂

Con el m√©todo grid en el objeto de eje, podemos activar y desactivar las l√≠neas de la cuadr√≠cula. Tambi√©n podemos personalizar el aspecto de las l√≠neas de la cuadr√≠cula utilizando los mismos argumentos de palabras clave que la funci√≥n plot:

fig, axes = plt.subplots(1, 2, figsize=(10,3))

# apariencia default de la grilla
axes[0].plot(x, x**2, x, x**3, lw=2)
axes[0].grid(True)

# apariencia de la grilla personalizada
axes[1].plot(x, x**2, x, x**3, lw=2)
axes[1].grid(color='b', alpha=0.5, linestyle='dashed', linewidth=0.5)

Lineas de eje¬∂

Tambi√©n podemos cambiar las propiedades de las lineas de los ejes:

fig, ax = plt.subplots(figsize=(6,2))

ax.spines['bottom'].set_color('blue')
ax.spines['top'].set_color('blue')

ax.spines['left'].set_color('red')

# sacamos las lineas del lado de la derecha
ax.spines['right'].set_color("none")
ax.yaxis.tick_left() # ponemos solos los tics de la izquierda

Ejes gemelos¬∂

A veces es √∫til tener ejes x o y duales en una figura; por ejemplo, al trazar curvas con diferentes unidades juntas. Matplotlib admite esto con las funciones twinx y twiny:

fig, ax1 = plt.subplots()

ax1.plot(x, x**2, lw=2, color="blue")
ax1.set_ylabel(r"area $(m^2)$", fontsize=18, color="blue")
for label in ax1.get_yticklabels():
    label.set_color("blue")
    
ax2 = ax1.twinx()
ax2.plot(x, x**3, lw=2, color="red")
ax2.set_ylabel(r"volume $(m^3)$", fontsize=18, color="red")
for label in ax2.get_yticklabels():
    label.set_color("red")

Ejes donde x e y son cero¬∂

fig, ax = plt.subplots()

Otros estilos de trama 2D¬∂

Adem√°s del m√©todo regular ‚Äúplot‚Äù, hay una serie de otras funciones para generar diferentes tipos de gr√°ficos. Consulte la galer√≠a de gr√°ficos de matplotlib para obtener una lista completa de los tipos de gr√°ficos disponibles. Algunos de los m√°s √∫tiles se muestran a continuaci√≥n:

Anotaci√≥n de texto¬∂

La anotaci√≥n de texto en las figuras de matplotlib se puede hacer usando la funci√≥n text. Admite el formateo LaTeX al igual que los textos y t√≠tulos de etiquetas de ejes:

Figuras de colores y contornos¬∂

Los mapas de colores y las figuras de contorno son √∫tiles para trazar funciones de dos variables. En la mayor√≠a de estas funciones, utilizaremos un mapa de colores para codificar una dimensi√≥n de los datos. Hay una serie de mapas de color predefinidos. Es relativamente sencillo definir mapas de colores personalizados. Para obtener una lista de mapas de colores predefinidos, consulte aqu√≠

alpha = 0.7
phi_ext = 2 * np.pi * 0.5

def flux_qubit_potential(phi_m, phi_p):
    return 2 + alpha - 2 * np.cos(phi_p) * np.cos(phi_m) - alpha * np.cos(phi_ext - 2*phi_p)

phi_m = np.linspace(0, 2*np.pi, 100)
phi_p = np.linspace(0, 2*np.pi, 100)
X,Y = np.meshgrid(phi_p, phi_m)
Z = flux_qubit_potential(X, Y).T

fig, ax = plt.subplots()
